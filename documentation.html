Lumina Wealth CRM

Comprehensive documentation for the Next-Gen AI Financial Advisor Platform.

v1.0-alpha Gemini 2.5 & 3.0 Integrated

1. Introduction

Lumina Wealth CRM is a specialized Customer Relationship Management tool designed for High Net-Worth Individual (HNI) Financial Advisors in India. Unlike generic CRMs, Lumina is built with a "Liquid Glass" aesthetic and is deeply integrated with Google's Gemini AI ecosystem to perform complex financial reasoning, real-time market research, and voice-based interactions.

The application serves as a frontend prototype with a fully functional "Brain" (AI) but relies on in-memory state and mock data for persistence in this version.

2. Tech Stack & Architecture

Frontend Core

  • React 19: Component-based architecture.
  • TypeScript: Strict type safety (Interfaces defined in types.ts).
  • Vite/Bundler: (Implied environment).

Styling & Animation

  • Tailwind CSS: Utility-first styling.
  • Framer Motion: Complex layout transitions, shared element animations, and micro-interactions.
  • Lucide React: Iconography.
  • Recharts: Financial data visualization.

Artificial Intelligence

  • @google/genai SDK: Direct integration.
  • Gemini 3.0 Pro Preview: For "Thinking" and complex reasoning.
  • Gemini 2.5 Flash: For "Search Grounding" and "Maps".
  • Gemini Live API: For real-time WebSocket audio streaming.

3. UI Design System

Lumina uses a distinct "Liquid Glass" design language.

  • Background: A complex, multi-layered radial gradient (`#020405` base) that creates depth.
  • GlassCard Component: The core building block (components/GlassCard.tsx). It uses backdrop-blur-xl, white transparency borders, and an internal "shine" layer using z-index isolation to create a premium feel.
  • Animations:
    • Staggered Entrance: Dashboard cards and list items slide in sequentially.
    • Hover/Tap: Buttons and cards scale slightly (`scale: 1.02`).
    • Layout Transitions: `AnimatePresence` handles page switching (`mode="wait"`).
  • Custom Components:
    • GlassDropdown.tsx: Replaces native OS selects with a themed, animated dropdown.
    • StatusBadge (Internal): Uses pulsing animations for "Active" status and rotating borders for "Onboarding".

11. Gemini Live (Voice)

File: components/LiveVoice.tsx

A real-time, low-latency voice conversational interface.

  • Protocol: WebSocket connection via ai.live.connect.
  • Audio Stack:
    • Input: AudioContext (16kHz) -> PCM conversion -> Base64 -> Stream to API.
    • Output: Base64 PCM (24kHz) from API -> Float32Array -> AudioBuffer -> Playback.
  • Visualizer: A pulsing, rotating orb that reacts to "Talking" and "Listening" states using Framer Motion.
  • Model: gemini-2.5-flash-native-audio-preview-09-2025.

13. Roadmap & Missing Features

To take this from a prototype to a production CRM, the following must be built:

Phase 1: Persistence

  • Implement Supabase or Firebase for user authentication and database.
  • Replace MOCK_CLIENTS with a useClients hook fetching from DB.

Phase 2: Real Integrations

  • Google OAuth: Allow users to sign in with Google to enable real Calendar and Drive access.
  • Email Integration: clicking an email address should actually log the email in the history via Gmail API.

Phase 3: AI Enhancements

  • RAG (Retrieval Augmented Generation): Allow the AI Advisor to "read" the PDF files in the Drive tab (e.g., "Summarize the policy document I just uploaded").
  • Function Calling: Allow the AI to perform actions, like "Schedule a meeting with Rahul" (actually creating the Calendar event).

Phase 4: Mobile Optimization

  • While responsive, the "Split View" layouts need a "Stack View" logic for mobile devices to avoid cramping.
